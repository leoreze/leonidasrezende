VittaCare Clinic Landing Page Template

Documentação do Template

Guia completo para instalar, editar, personalizar e publicar a landing page premium para clínicas, consultórios e negócios de saúde.

1. O que está incluso

  • Landing page completa em HTML5 semântico.
  • CSS moderno com variáveis, grid responsivo, componentes e animações.
  • JavaScript puro para menu mobile, FAQ, tabs, modal, validação e toast.
  • Assets SVG editáveis: logo, mockup e capa OG.
  • Documentação em HTML e README.md.
  • Copy comercial para Behance, Patreon, Gumroad e marketplaces.

2. Estrutura de pastas

/index.html
/assets/css/style.css
/assets/js/main.js

3. Como abrir o projeto

Você pode abrir o arquivo index.html diretamente no navegador. Para uma visualização mais fiel, use um servidor local.

npx serve .
# ou
python -m http.server 8000

4. Como trocar textos

Todos os textos principais ficam no arquivo index.html. Procure pelas seções:

  • <section class="hero"> para título principal, subtítulo e CTAs.
  • #beneficios para cards de benefícios.
  • #especialidades para tabs e serviços.
  • #planos para preços, pacotes e ofertas.
  • #faq para perguntas frequentes.

5. Como trocar cores

Abra assets/css/style.css e edite as variáveis no bloco :root.

VariávelUso
--color-primaryCor principal da marca.
--color-secondaryCor de destaque para botões e elementos visuais.
--color-accentCor complementar para gradientes e detalhes.
--color-bgFundo geral da página.
--color-textTexto principal.
--color-mutedTextos secundários.

6. Como trocar imagens

Substitua os arquivos em assets/img/ mantendo o mesmo nome, ou altere os caminhos no HTML:

  • logo-mark.svg — símbolo da marca.
  • hero-mockup.svg — mockup visual do hero.
  • og-cover.svg — imagem de compartilhamento e capa comercial.

7. Componentes reutilizáveis

Classes principais:

  • .btn, .btn-primary, .btn-secondary, .btn-ghost
  • .section, .section-header, .container, .grid
  • .card, .benefit-card, .feature-card, .pricing-card
  • .badge, .tag, .faq-item, .cta-box
  • .tabs, .modal, .toast

8. Como duplicar ou remover seções

Para duplicar, copie o bloco completo de uma <section> no index.html. Para remover, apague a seção e remova o respectivo link do menu, se existir.

Mantenha IDs únicos. Exemplo: se duplicar id="planos", altere para id="pacotes".

9. JavaScript

O arquivo assets/js/main.js contém funções separadas e comentadas para:

  • Menu mobile abre/fecha.
  • Header com sombra ao rolar.
  • Animação de entrada com Intersection Observer.
  • FAQ accordion.
  • Tabs de especialidades.
  • Modal de apresentação.
  • Máscara simples de telefone.
  • Validação de formulário e toast.

10. Publicação

GitHub Pages

  1. Crie um repositório.
  2. Envie todos os arquivos.
  3. Ative Pages em Settings → Pages.

Netlify ou Vercel

  1. Faça upload da pasta ou conecte o repositório.
  2. Não precisa de build command.
  3. Diretório de publicação: raiz do projeto.

Render

Use Static Site e defina a pasta raiz como diretório de publicação.
