Nexora LeadFlow — Landing Page Comercial Premium

Este template foi criado para agências, consultorias, freelancers, SaaS, negócios locais e criadores que precisam de uma página comercial moderna, responsiva e pronta para captação de leads.

Stack utilizada: HTML5 semântico, CSS3 moderno e JavaScript puro. Não há dependências obrigatórias além das fontes do Google Fonts, que podem ser removidas se preferir usar fontes locais ou do sistema.

Como customizar

Trocar textos

Abra index.html e edite os textos diretamente nas seções. Procure por IDs como #hero, #beneficios, #features, #planos e #faq.

Trocar cores

No topo do style.css, altere as variáveis:

:root {
  --color-primary: #6d5ef7;
  --color-secondary: #00d4ff;
  --color-accent: #ffb86b;
  --color-bg: #070816;
}

Trocar imagens

Substitua os arquivos em /assets/img/ mantendo os mesmos nomes, ou altere o caminho da imagem no HTML. Para mockups, use imagens em SVG, PNG ou WebP.

Alterar CTAs

Procure pelos botões com classes .btn-primary e .btn-secondary. Troque o texto e o atributo href para WhatsApp, formulário, checkout ou agenda.

Componentes reutilizáveis

  • .btn, .btn-primary, .btn-secondary, .btn-ghost: botões.
  • .card, .benefit-card, .feature-card, .pricing-card: cards.
  • .badge, .tag: etiquetas e microcopy.
  • .grid, .cards-grid, .feature-grid: grids responsivos.
  • .faq-item: accordion de perguntas frequentes.
  • [data-tabs]: tabs controladas pelo JavaScript.
  • [data-modal]: modal simples para vídeo, oferta ou captura.
  • [data-toast]: alerta visual para feedback do formulário.

Duplicar uma seção

Copie o bloco completo da seção no index.html, altere o ID e atualize textos, cards e links.

Remover uma seção

Remova o bloco <section> correspondente e, se houver link no menu, remova também o item em <nav>.

Como publicar

Hospedagem simples

Envie todos os arquivos para a pasta pública do servidor. O arquivo principal deve continuar como index.html.

GitHub Pages

  1. Crie um repositório no GitHub.
  2. Faça upload dos arquivos do template.
  3. Acesse Settings → Pages.
  4. Selecione a branch principal e a pasta root.

Netlify, Vercel ou Render

Importe o repositório ou arraste a pasta do projeto. Como é um site estático, não precisa de build command.

Checklist antes de vender

  • Testar versão mobile, tablet e desktop.
  • Validar todos os links e CTAs.
  • Trocar imagens placeholder por mockups finais.
  • Revisar textos comerciais e metatags SEO.
  • Compactar arquivos em ZIP.
  • Criar preview no Behance com capa, seções e detalhes dos componentes.
  • Definir licença: uso pessoal, comercial simples ou comercial estendida.
  • Publicar em Gumroad, Creative Market, Lemon Squeezy ou similar.
