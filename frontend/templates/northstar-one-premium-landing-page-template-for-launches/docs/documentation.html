1. Como instalar e abrir

Não há dependências obrigatórias. Basta baixar o ZIP, extrair a pasta e abrir o arquivo index.html no navegador.

northstar-one-landing-template/
├── index.html
├── assets/css/style.css
├── assets/js/main.js
├── assets/img/
├── docs/documentation.html
└── README.md

Para editar, use VS Code, Cursor, WebStorm ou qualquer editor de texto.

2. Estrutura de arquivos

  • index.html: página principal e conteúdo editável.
  • style.css: variáveis, layout, responsividade e componentes.
  • main.js: menu mobile, tabs, accordion, modal, toast, formulário e animações.
  • assets/img: logo, mockup SVG e imagem OG.
  • docs/documentation.html: documentação visual.
  • README.md: guia rápido + conteúdo comercial.

3. Como personalizar

Trocar textos

Abra index.html e edite títulos, descrições, CTAs, depoimentos, preços e perguntas frequentes.

Trocar cores

No início do style.css, altere as variáveis:

:root {
  --color-primary: #0D1321;
  --color-secondary: #1F6FEB;
  --color-accent: #D6B268;
  --color-bg: #F4EFE6;
}

Trocar imagens

Substitua os SVGs em assets/img ou aponte as tags <img> para novas imagens.

Alterar CTAs

Procure por Get the template, Comprar template, Ver preview e edite os textos ou links.

4. Componentes reutilizáveis

Accordion acessível para FAQ.

Cards de preço com plano destacado.

Formulário com validação simples.
.modal
Modal opcional para preview.
.toast
Alerta visual para feedback.

5. Como publicar

GitHub Pages

Suba os arquivos para um repositório e ative Pages em Settings → Pages.

Netlify

Arraste a pasta do template para o painel do Netlify ou conecte o repositório.

Vercel

Importe o repositório. Como o projeto é estático, não precisa de build command.

Render

Use Static Site e defina a pasta raiz como diretório público.
